# Backlog Organizer - Output Templates

## Standard Backlog Organization Output

```
### Backlog Organization - [Project / Area] - [Date]

**Total Items Reviewed:** [N]
**Framework Applied:** [Name]

#### Recommended Backlog Order

| Rank | Ticket / Item | Status | Score / Rationale | Issues Found |
|---|---|---|---|---|

#### Items Recommended for Removal or Deferral

| Item | Reason (Stale / Duplicate / Out of Scope) | Recommended Action |
|---|---|---|

#### Items Needing More Definition Before They Can Be Ranked

| Item | What's Missing |
|---|---|

#### Suggested Next Steps
- [Action 1]
- [Action 2]
```

## Notes on Template Use

- "Issues Found" column should note Stale / Duplicate / Vague / Out of Scope flags inline
- "Recommended for Removal or Deferral" is required - if nothing qualifies, state that explicitly
- "Needs More Definition" surfaces items the PM must fix before they can be groomed - do not silently drop them into the ranked list
- Suggested Next Steps should be concrete and actionable (e.g., "Schedule 30-min definition session for [item]", "Merge [A] into [B] and close [A]")
